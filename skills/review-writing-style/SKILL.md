# Workflow: Review Writing Style

## Goal

Produce concise technical communication for reviews, tickets, and internal engineering discussions.

## Rules

- Prefer direct, short sentences.
- State findings before summaries.
- Use specific references (`file:line`, commit IDs, command output summary).
- Avoid filler, marketing language, and vague wording.
- Ask clear decision questions only when needed.

## Exclusions

- Do not apply this strict terse style to user-facing documentation.
