# Workflow: Thinking Root Cause

## Goal

Find fundamental causes of failures and unexpected behavior using structured 5-why analysis.

## Process

1. Gather symptoms, environment context, and reproduction path.
2. Build 5-level why chain from symptom to root cause.
3. Validate each step with logs, code, config, or execution evidence.
4. Produce root cause statement plus targeted remediation options.
