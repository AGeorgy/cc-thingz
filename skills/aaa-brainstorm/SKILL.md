# Workflow: Brainstorm

## Goal

Turn an idea into a validated implementation direction before coding.

## Steps

1. Gather local context from code, docs, and current branch state.
2. Ask focused questions one at a time.
3. Propose 2-3 approaches with trade-offs and one recommended option.
4. Validate chosen direction with the user.
5. Offer next action:
   - Create implementation plan using `$planning-make`
   - Start implementation directly

## Guidance

- Favor simplicity and clear boundaries.
- Avoid speculative features.
- Use concrete examples tied to existing project patterns.
